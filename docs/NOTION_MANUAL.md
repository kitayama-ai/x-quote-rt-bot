# X（Twitter）AI引用RT自動化ダッシュボード 操作マニュアル

> 最終更新: 2026-02-22 | バージョン: v2.0

---

## このツールについて

このツールは、**海外のAI系バズツイートを自動で見つけて、AIが日本語コメントを生成し、あなたのXアカウントから引用RT（リツイート）として投稿する**自動化ツールです。

すべての操作はブラウザ上のダッシュボードから行えます。プログラミングの知識は一切必要ありません。

### ツールの流れ（全体像）

```
1. 海外のバズツイートを自動で収集
         |
2. AIが日本語コメントを8パターンから自動生成
         |
3. あなたが承認 → 投稿（半自動モード）
   または 全自動で投稿（完全自動モード）
```

### ダッシュボードURL

**https://isai-11f7b.web.app**

> このURLをブックマークしておくと便利です。スマートフォンのブラウザからもアクセスできます。

## 1. ログイン方法

ダッシュボードにアクセスすると、ログイン画面が表示されます。

[スクリーンショット: ログイン画面（Xロゴ + 「X でログイン」ボタン）]

### 手順

1. ダッシュボードURL（ https://isai-11f7b.web.app ）をブラウザで開く
2. **「X でログイン」** ボタンをクリック
3. Xのログイン画面がポップアップで表示される
4. お使いのXアカウントでログインし、アプリを許可する
5. 自動的にダッシュボードに戻り、あなた専用の画面が表示される

> **ポイント**: ログインにはX（Twitter）アカウントを使用します。Googleアカウントでのログインには対応していません。

> **安心ポイント**: すべてのデータ（APIキー、キュー、設定など）はログインアカウントごとに完全に分離されています。他のユーザーがあなたのデータを見ることはできません。

### ログインできない場合

| 症状 | 確認すること |
|---|---|
| ポップアップが表示されない | ブラウザのポップアップブロックを解除してください |
| 「認証中...」のまま進まない | ページを再読み込みして、もう一度お試しください |
| エラーメッセージが表示される | X Developer Portalのコールバック URL に `https://isai-11f7b.firebaseapp.com/__/auth/handler` が登録されているか確認してください |

## 2. ダッシュボードの画面構成

ログイン後、以下の画面が表示されます。

[スクリーンショット: ダッシュボード全体画面]

### 画面上部: KPI（数値カード）

ログイン直後に表示される5つのカードで、現在の状況がひと目で分かります。

| カード | 意味 |
|---|---|
| **承認待ち** | まだ確認していないツイートの件数 |
| **承認済み** | 承認したが、まだ投稿されていないツイートの件数 |
| **投稿済み（本日）** | 今日投稿された引用RTの件数 |
| **投稿済み（累計）** | これまでに投稿した引用RTの合計件数 |
| **スキップ** | スキップ（投稿しない判断をした）ツイートの件数 |

### クイックアクションバー

KPIカードの下に表示されるボタン列です。よく使う操作をワンクリックで実行できます。

| ボタン | 動作 |
|---|---|
| **収集** | 海外バズツイートを今すぐ収集 |
| **生成** | AIコメントを今すぐ生成 |
| **投稿** | 引用RTを今すぐXに投稿 |
| **更新** | ダッシュボードのデータを最新に更新 |

### タブ（メインメニュー）

画面のメイン領域は8つのタブに分かれています。タブをクリックして切り替えます。

| タブ名 | 何ができるか | 使用頻度 |
|---|---|---|
| **キュー一覧** | 収集されたツイートの確認・承認・スキップ | 毎日使う |
| **投稿履歴** | 投稿済みの引用RTを時系列で確認 | 時々確認 |
| **メトリクス** | 投稿数やいいね数のグラフで推移を確認 | 週1回程度 |
| **選定設定** | AIの生成スタイルやトピックのカスタマイズ | 必要な時だけ |
| **PDCA分析** | 承認率やスキップ理由の統計で改善ポイントを把握 | 週1回程度 |
| **操作** | 手動操作ボタン・ツイートURL手動追加 | 必要な時だけ |
| **設定** | APIキー登録・収集条件調整・運用モード切替 | 初回設定時 |
| **使い方** | ツール内のヘルプガイド | 困った時 |

## 3. 各タブの詳細説明

### 3-1. キュー一覧タブ

[スクリーンショット: キュー一覧タブ（テーブルとフィルターボタン）]

**最も頻繁に使うタブです。** 収集されたツイートの一覧が表示され、ここで承認やスキップの判断を行います。

#### テーブルの見方

| 列名 | 内容 |
|---|---|
| ステータス | 現在の状態（承認待ち / 承認済み / 投稿済み / スキップ） |
| 元ツイート | 収集された海外ツイートの冒頭部分 |
| 生成テキスト | AIが生成した日本語コメント（引用RT文面） |
| エンゲージメント | 元ツイートのいいね・RT・リプライ数 |
| マッチ | 収集条件との合致度 |
| 追加日時 | ツイートがキューに追加された日時 |
| アクション | 承認 / スキップボタン |

#### フィルター機能

テーブル上部のフィルターボタンで、表示するツイートを絞り込めます。

- **すべて** — 全件表示
- **承認待ち** — まだ確認していないツイートのみ
- **承認済み** — 承認したツイートのみ
- **スキップ** — スキップしたツイートのみ

#### 詳細モーダル（ポップアップ）

テーブルの行をクリックすると、詳細画面がポップアップで開きます。

[スクリーンショット: 詳細モーダル（元ツイート・日本語要約・AIコメント表示）]

詳細モーダルに表示される情報:

| 項目 | 内容 |
|---|---|
| 投稿者情報 | 元ツイートの投稿者名・アカウント名 |
| エンゲージメント | いいね・RT・リプライ数 |
| 元ツイート（原文） | 英語の元ツイート全文 |
| 日本語要約 | Gemini AIが翻訳・要約した日本語テキスト |
| AI生成コメント | 実際にXに投稿される日本語コメント文面 |
| 追加日時・ソース・テンプレート | メタ情報 |

モーダル下部の **「承認」** または **「スキップ」** ボタンで判断します。

> **ヒント**: 日本語要約が表示されない場合は、「設定」タブでGemini API Keyが登録されているか確認してください。

### 3-4. 選定設定タブ

[スクリーンショット: 選定設定タブ（各種入力フォーム）]

AIの生成スタイルや、どのようなツイートを収集するかをカスタマイズするタブです。細かくコントロールしたい方向けの設定が揃っています。

#### 今週のフォーカス

| 設定項目 | 説明 | 入力例 |
|---|---|---|
| フォーカスディレクティブ | 今週の方針を自由記述 | 「今週はAIエージェント関連のツイートを中心に」 |
| フォーカスキーワード | 注目したいキーワード（カンマ区切り） | MCP, tool use, AI agent |
| フォーカスアカウント | 注目したいアカウント（カンマ区切り） | @sama, @karpathy |

#### トピック設定

| 設定項目 | 説明 | 入力例 |
|---|---|---|
| 優先トピック | 優先的に収集したいジャンル | AI agents, product launches |
| 回避トピック | 避けたいジャンル | crypto, politics |

#### アカウント設定

| 設定項目 | 説明 | 入力例 |
|---|---|---|
| 優先アカウント（ブースト） | 優先度を上げたいアカウント（スコア1.5倍） | sama, karpathy |
| ブロックアカウント | 完全に除外したいアカウント | spammer123 |

#### キーワードウェイト

特定キーワードを含むツイートの優先度を数値で調整できます。数値が高いほど優先されます。

#### 引用RTプロンプト設定

AIが生成するコメントのスタイルを細かく設定できます。

| 設定項目 | 説明 | 入力例 |
|---|---|---|
| ペルソナ名 | AIが演じるキャラクター名 | NinjaGuild |
| 一人称 | コメントで使う一人称 | 僕 / 私 / 俺 |
| ポジション | 自分の立場・何者か | AI活用の第一人者 |
| 差別化ポイント | 他のアカウントとの違い | 海外ニュースを即座に要点整理 |
| トーン | 文章の雰囲気 | カジュアルで熱量高め |
| 口調パターン | 文体の具体的なパターン | 体言止め、箇条書き、感情語 |
| NGワード | 使ってはいけない単語（カンマ区切り） | 素晴らしい, 革新的 |
| カスタム指示 | AIへの追加指示（自由記述） | 具体的な数字を必ず含めること |

#### テンプレート（8パターン）

AIが自動で最適なテンプレートを選びます。使いたくないパターンはチェックを外してOFFにできます。

| テンプレート名 | 内容 |
|---|---|
| 市場インパクト型 | 市場/投資家への影響を箇条書きで解説 |
| 要点まとめ型 | ポイント3つを速報形式でまとめ |
| 警告・問題提起型 | 根拠やデータをもとに警鐘を鳴らす |
| 激震分析型 | 業界への影響を断定的に分析 |
| 衝撃速報型 | 歴史的意義を断言するスタイル |
| 独占入手型 | スクープ情報を断定的に伝える |
| ダーク警告型 | リスクや危険性を冷徹にレポート |
| 伝説・歴史型 | 歴史的偉業をドラマチックに表現 |

> **ヒント**: 8つすべてを有効にしておくと、1日に複数投稿しても毎回違うスタイルのコメントが生成されます。

設定を変更したら、画面下部の **「選定設定を保存」** ボタンを忘れずにクリックしてください。

### 3-6. 操作タブ

[スクリーンショット: 操作タブ（4つの実行ボタンとURL入力欄）]

手動で操作を実行するためのタブです。通常は自動で動きますが、「今すぐ実行したい」場合にここから操作できます。

#### 手動操作ボタン

| ボタン | 動作 | 実行タイミングの目安 |
|---|---|---|
| **今すぐ収集** | 海外バズツイートを即座に収集する | 新しいツイートを取り込みたい時 |
| **今すぐ生成** | 承認済みツイートにAIコメントを生成する | 承認した直後にコメントを作りたい時 |
| **今すぐ投稿** | 生成済みの引用RTをXに投稿する | すぐに投稿したい時 |
| **今すぐ更新** | ダッシュボードのデータを最新に更新する | データが古いと感じた時 |

> ボタンを押すと操作リクエストが送信されます。通常5〜10分以内に自動実行されます。

#### ツイートURL手動追加

自動収集以外に、自分で見つけたツイートを引用RTしたい場合に使います。

**手順:**

1. 「操作」タブを開く
2. 「ツイートURLを手動追加」セクションの入力欄にURLを貼り付ける
   - 形式: `https://x.com/username/status/XXXXXXXXX`
3. 「追加」ボタンをクリック
4. 「今すぐ生成」ボタンでAIコメントを生成
5. 「今すぐ投稿」ボタンでXに投稿

#### 操作リクエスト履歴

過去に実行した操作リクエストの履歴が表示されます。処理状態（処理中 / 完了 / エラー）を確認できます。

### 3-7. 設定タブ

[スクリーンショット: 設定タブ（APIキー入力フォーム・運用モード選択）]

初回に必ず設定が必要なタブです。APIキーの登録、収集条件の調整、運用モードの切り替えを行います。

#### アカウント情報

ログイン中のアカウント名とアイコンが表示されます。

#### APIキー設定

以下のキーを登録してください（詳しい取得方法は後述の「4. API設定の方法」を参照）。

| キー名 | 必須/任意 | 用途 |
|---|---|---|
| **Gemini API Key** | 必須 | AIコメント生成・日本語要約 |
| **Discord Webhook URL** | 任意 | 投稿通知をDiscordに送信 |
| **X API Bearer Token** | 必須 | バズツイートの収集（検索） |
| **X API Key (Consumer Key)** | 必須 | X認証（投稿用） |
| **X API Secret (Consumer Secret)** | 必須 | X認証（投稿用） |
| **X Access Token** | 必須 | X投稿の実行 |
| **X Access Token Secret** | 必須 | X投稿の実行 |

入力後、**「保存」** ボタンをクリックしてください。各キーの横に「設定済み」と表示されれば登録完了です。

> **安心ポイント**: 登録したキーはFirestoreに安全に保存され、あなた自身しかアクセスできません。

#### 収集設定

バズツイートの収集条件を調整できます。

| 設定項目 | 説明 | おすすめ値 |
|---|---|---|
| 1クエリあたり最大取得件数 | 1回の収集で取得するツイート数（10〜100） | 50 |
| 最低いいね数 | この数以上のいいねがあるツイートのみ収集 | 500 |
| 最大経過時間 | 投稿からこの時間以内のツイートのみ収集 | 48時間（2日） |

> **コスト目安**: 取得件数を増やすとAPI利用料が微増しますが、30件程度の候補を出すなら100推奨です。

#### 運用モード

投稿までの流れを選択します。

| モード | 動作 | おすすめの人 |
|---|---|---|
| **半自動モード（承認制）** | AIが生成 → あなたが承認 → 投稿 | 最初はこちらがおすすめ。内容を確認してから投稿したい方 |
| **完全自動モード** | AIが生成 → 自動承認 → 自動投稿 | 慣れてきたら切り替え。手間ゼロで運用したい方 |

モードはいつでも切り替えられます。変更は即座に反映されます。

## 4. API設定の方法（初回のみ）

ダッシュボードを使い始めるために、2つのAPIキーの取得が必要です。

### 4-1. Gemini API Key の取得

Gemini APIは、AIコメントの生成と日本語要約に使用します。

**手順:**

1. ブラウザで **[Google AI Studio](https://aistudio.google.com/apikey)** を開く
2. Googleアカウントでログインする
3. 「Get API key」または「APIキーを作成」をクリック
4. 表示されたAPIキー（`AIzaSy...` で始まる文字列）をコピーする
5. ダッシュボードの **「設定」タブ → Gemini API Key** 欄に貼り付ける
6. **「保存」** ボタンをクリック

[スクリーンショット: Google AI Studio のAPIキー作成画面]

> **費用**: Gemini APIはほぼ無料で使えます（月額$0〜0.50程度）。

---

### 4-2. X（Twitter）API キーの取得

X APIは、バズツイートの収集とXへの投稿に使用します。合計5つのキーが必要です。

#### ステップ 1: X Developer Portal にアクセス

1. ブラウザで **[X Developer Portal](https://developer.x.com/en/portal/dashboard)** を開く
2. Xアカウントでログインする
3. Developer アカウントの申請がまだの場合は、画面の指示に従って申請する

[スクリーンショット: X Developer Portal トップ画面]

#### ステップ 2: アプリの認証設定

1. Developer Portal で対象のアプリを選択する
2. 右パネルの **「ユーザー認証設定」** → **「セットアップ」** をクリック
3. 以下を設定する:

| 設定項目 | 入力する値 |
|---|---|
| アプリの権限 | **「読み取りと書き込み」** |
| アプリの種類 | **「ウェブアプリ、自動化アプリまたはボット」** |
| Callback URI | `https://isai-11f7b.firebaseapp.com/__/auth/handler` |
| Website URL | `https://isai-11f7b.web.app` |
| 利用規約URL | `https://isai-11f7b.web.app/terms.html` |
| ポリシーURL | `https://isai-11f7b.web.app/privacy.html` |

4. **「変更する」** をクリックして保存

[スクリーンショット: X Developer Portal ユーザー認証設定画面]

> **注意**: 「組織名」フィールドにURLを求められることがありますが、これはUIの翻訳バグです。任意のURL（ポートフォリオURLなど）を入力してください。

#### ステップ 3: APIキーの取得

1. Developer Portal でアプリの **「Keys and Tokens」** セクションを開く
2. 以下の5つのキーを取得する:

| キー名 | 取得方法 | ダッシュボードの入力先 |
|---|---|---|
| **Bearer Token** | 「Bearer Token」セクションの「取り込み」または「生成」 | X API Bearer Token |
| **API Key (Consumer Key)** | 「Consumer Keys」セクション | X API Key |
| **API Secret (Consumer Secret)** | 「Consumer Keys」セクション | X API Secret |
| **Access Token** | 「Authentication Tokens」セクションの「生成」 | X Access Token |
| **Access Token Secret** | 「Authentication Tokens」セクションの「生成」 | X Access Token Secret |

[スクリーンショット: X Developer Portal Keys and Tokens画面]

> **重要**: Access Token と Access Token Secret は生成時に一度だけ表示されます。必ずその場でコピーしてください。画面を閉じると二度と表示されません（再生成は可能です）。

> **重要**: アプリの権限を「読み取りのみ」から「読み取りと書き込み」に変更した場合は、Access Token を再生成する必要があります。

#### ステップ 4: ダッシュボードに登録

1. ダッシュボードの **「設定」タブ** を開く
2. 「APIキー設定」セクションの各入力欄に、取得した5つのキーを貼り付ける
3. **「保存」** ボタンをクリック
4. 各キーの横に **「設定済み」** と表示されれば完了

### 4-3. Discord Webhook URL の取得（任意）

投稿通知をDiscordで受け取りたい場合に設定します。不要であればスキップしてください。

**手順:**

1. Discordで通知を受け取りたいチャンネルの設定を開く
2. **「連携サービス」** → **「ウェブフック」** を選択
3. **「新しいウェブフック」** を作成
4. 表示されたURLをコピー
5. ダッシュボードの **「設定」タブ → Discord Webhook URL** 欄に貼り付ける
6. **「保存」** をクリック

## 5. 投稿の流れ（ワークフロー）

### 5-1. 半自動モード（承認制）の場合

最も安全な運用方法です。最初はこちらを推奨します。

```
ステップ 1: 収集（自動 or 手動）
    海外バズツイートが自動収集される
    （または「操作」タブの「今すぐ収集」で手動実行）
              |
ステップ 2: 確認・承認
    「キュー一覧」タブで収集されたツイートを確認
    → 行をクリックして詳細を見る
    → 「承認」または「スキップ」を選択
              |
ステップ 3: AIコメント生成（自動 or 手動）
    承認されたツイートに対してAIがコメントを自動生成
    （または「操作」タブの「今すぐ生成」で手動実行）
              |
ステップ 4: 投稿（自動 or 手動）
    生成されたコメント付きで引用RTがXに投稿される
    （または「操作」タブの「今すぐ投稿」で手動実行）
              |
ステップ 5: 確認
    「投稿履歴」タブで投稿内容を確認
```

#### 日常の操作（半自動モード）

1. ダッシュボードを開く
2. **「キュー一覧」タブ** で承認待ちのツイートを確認
3. 行をクリックして詳細モーダルを開く
4. 元ツイート（英語）と日本語要約を読んで内容を把握
5. AI生成コメント（投稿される文面）を確認
6. 問題なければ **「承認」** をクリック
7. 投稿したくない場合は **「スキップ」** をクリック

> **ヒント**: スキップの際に理由（的外れ、重複、低品質など）を選択できます。この情報はPDCA分析に活用され、収集精度が自動で改善されていきます。

---

### 5-2. 完全自動モードの場合

慣れてきたら完全自動モードに切り替えると、手間ゼロで運用できます。

```
ステップ 1: 収集（自動）
              |
ステップ 2: 自動承認
              |
ステップ 3: AIコメント生成（自動）
              |
ステップ 4: 投稿（自動）
```

基本的にあなたが操作する必要はありません。定期的にダッシュボードを開いて投稿状況を確認するだけでOKです。

---

### 5-3. 自動実行スケジュール

設定済みのスケジュールに従って、以下が自動で動いています。

| 時刻（日本時間） | 動作内容 |
|---|---|
| 毎朝 6:00 | バズツイートの自動収集 |
| 毎朝 6:30 | AIコメントの自動生成 |
| 7:00 / 8:30 / 10:15 / 12:00 / 14:15 / 16:00 / 18:00 / 19:45 / 21:00 / 22:30 | 引用RTの自動投稿（1日最大10回） |
| 毎晩 23:00 | エンゲージメント（いいね・RT数）の計測 |
| 毎週月曜 | 週次レポートの自動生成 |

> 自動スケジュールとは別に、「操作」タブからいつでも手動実行できます。

## 6. 手動操作の使い方

### 6-1. バズツイートを今すぐ収集する

1. **「操作」タブ** を開く
2. **「今すぐ収集」** ボタンをクリック
3. 画面に「リクエスト送信済み」と表示される
4. 5〜10分後に収集が完了する
5. **「キュー一覧」タブ** で結果を確認

### 6-2. AIコメントを今すぐ生成する

1. 承認済みのツイートがあることを確認（「キュー一覧」タブ）
2. **「操作」タブ** を開く
3. **「今すぐ生成」** ボタンをクリック
4. 5〜10分後に生成が完了
5. **「キュー一覧」タブ** で生成されたコメントを確認

### 6-3. 引用RTを今すぐ投稿する

1. AI生成済みのツイートがあることを確認
2. **「操作」タブ** を開く
3. **「今すぐ投稿」** ボタンをクリック
4. 5〜10分後にXに投稿される
5. **「投稿履歴」タブ** で投稿結果を確認

### 6-4. ダッシュボードを今すぐ更新する

1. **「操作」タブ** を開く
2. **「今すぐ更新」** ボタンをクリック
3. 数分後にダッシュボードのデータが最新になる

### 6-5. 特定のツイートを手動追加して投稿する

自動収集とは別に、自分で見つけた特定のツイートを引用RTしたい場合の手順です。

1. Xで引用RTしたいツイートを見つけ、URLをコピーする
   - 形式: `https://x.com/ユーザー名/status/数字`
2. ダッシュボードの **「操作」タブ** を開く
3. **「ツイートURLを手動追加」** セクションにURLを貼り付ける
4. **「追加」** ボタンをクリック
5. **「今すぐ生成」** ボタンでAIコメントを生成
6. **「キュー一覧」タブ** で内容を確認・承認
7. **「今すぐ投稿」** ボタンでXに投稿

## 9. よくある質問（FAQ）

### Q. ダッシュボードにデータが表示されません

**A.** 「操作」タブの **「今すぐ更新」** ボタンをクリックしてください。数分後にデータが表示されます。

---

### Q. 投稿が止まっているようです

**A.** 以下を確認してください:

1. 「設定」タブでAPIキーが正しく登録されているか（「設定済み」と表示されているか）
2. X APIのクレジット残高があるか
   - 残高確認: [console.x.com](https://console.x.com) → Billing
3. GitHub Actionsのワークフローがエラーになっていないか

---

### Q. 日本語要約が表示されません

**A.** 「設定」タブで **Gemini API Key** が登録されているか確認してください。キーが未登録だと日本語要約は表示されません。

---

### Q. AIが生成するコメントの雰囲気を変えたいです

**A.** 「選定設定」タブの **「引用RTプロンプト設定」** セクションで以下を調整できます:

- **トーン**: 文章の雰囲気を変更（例: 「カジュアルで親しみやすい」「専門的で分析的」）
- **ペルソナ名**: 投稿者キャラクターの名前
- **NGワード**: 使ってほしくない表現を指定
- **カスタム指示**: AIへの追加指示を自由に記述

変更後は **「選定設定を保存」** ボタンを忘れずにクリックしてください。変更は次回のコメント生成から反映されます。

---

### Q. 特定のトピックだけ収集したいです

**A.** 「選定設定」タブで以下を設定してください:

- **今週のフォーカス**: 注目テーマを記述
- **優先トピック**: 収集したいジャンルをカンマ区切りで入力
- **回避トピック**: 避けたいジャンルをカンマ区切りで入力

---

### Q. 半自動モードと完全自動モードの違いは何ですか？

**A.**

| 項目 | 半自動モード | 完全自動モード |
|---|---|---|
| 収集 | 自動 | 自動 |
| 承認 | **あなたが手動で判断** | 自動承認 |
| コメント生成 | 自動 | 自動 |
| 投稿 | スケジュールに従い自動 | スケジュールに従い自動 |
| メリット | 内容を確認してから投稿できて安心 | 完全に手間ゼロ |
| デメリット | 定期的にダッシュボードを確認する必要がある | 意図しない投稿がされる可能性がある |

> 最初は半自動モードで運用し、AIの生成品質に満足できたら完全自動モードに切り替えるのがおすすめです。

---

### Q. 運用モードを切り替えたいです

**A.** 「設定」タブの **「運用モード」** セクションで、希望するモードのラジオボタンを選択してください。変更は即座に反映されます。

---

### Q. 投稿頻度を変更したいです

**A.** 投稿スケジュールはGitHub Actionsで管理されています。投稿頻度の変更が必要な場合は、担当開発者にご相談ください。

## 10. トラブルシューティング

### エラー別の対処法

| エラー / 症状 | 原因 | 対処法 |
|---|---|---|
| **403 Forbidden** | X APIのスパム判定 | 30分ほど待ってから再実行してください |
| **402 Payment Required** | X APIのクレジット残高が0 | [console.x.com](https://console.x.com) → Billing でクレジットを購入してください |
| **401 Unauthorized** | アクセストークンの期限切れ | 「設定」タブでX API のAccess Token / Access Token Secretを再登録してください |
| **ダッシュボードにデータが表示されない** | データ未生成 | 「操作」タブの「今すぐ更新」をクリックしてください |
| **日本語要約が表示されない** | Gemini API Key未登録 | 「設定」タブでGemini API Keyを登録してください |
| **ログインできない** | コールバックURL未登録 | X Developer Portalで `https://isai-11f7b.firebaseapp.com/__/auth/handler` がCallback URIに登録されているか確認してください |
| **AIコメントが生成されない** | 承認済みツイートがない | 「キュー一覧」タブでツイートを承認してから「今すぐ生成」を実行してください |
| **投稿されない** | 生成済みツイートがない | 先に「今すぐ生成」を実行してAIコメントを生成してから「今すぐ投稿」を実行してください |

### それでも解決しない場合

ツールに関する質問やトラブルは、担当開発者までお問い合わせください。

## 用語集

| 用語 | 意味 |
|---|---|
| **引用RT（引用リツイート）** | 他の人のツイートに自分のコメントを添えてシェアする機能 |
| **バズツイート** | 多くのいいね・RTがついて話題になっているツイート |
| **キュー** | 収集されたツイートが順番に並んでいる待ち行列 |
| **承認** | 収集されたツイートを「投稿してOK」と判断すること |
| **スキップ** | 収集されたツイートを「投稿しない」と判断すること |
| **ペルソナ** | AIが演じるキャラクター設定（名前・口調・立場など） |
| **テンプレート** | AIコメントの生成パターン（8種類） |
| **APIキー** | 外部サービスに接続するための認証情報（パスワードのようなもの） |
| **Gemini API** | Googleが提供するAIサービス。コメント生成と日本語要約に使用 |
| **X API** | X（Twitter）が提供するサービス。ツイートの収集と投稿に使用 |
| **Bearer Token** | X APIでツイートを検索・収集するための認証キー |
| **Access Token** | X APIでツイートを投稿するための認証キー |
| **PDCA** | Plan（計画）Do（実行）Check（確認）Action（改善）の略。改善サイクルのこと |
| **GitHub Actions** | 自動実行スケジュールを管理する仕組み |
| **Firebase / Firestore** | データの保存・ホスティングに使用しているGoogleのサービス |
| **Discord Webhook** | Discordチャンネルに通知を送る仕組み |
