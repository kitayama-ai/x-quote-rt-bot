## 4. リスク管理 ★

### 4-1. X利用規約リスク

| リスク                         | 対策                                                        | 重要度    |
| ------------------------------ | ----------------------------------------------------------- | --------- |
| **アカウント凍結**             | 新規アカウント作成後、3-5日は手動投稿で暖機運転             | 🔴 最重要 |
| **複数アカウント検出（将来）** | 各アカウントに固有メール・電話番号。API投稿でブラウザ不使用 | 🔴 最重要 |
| **自動投稿の検出**             | 投稿時間をランダムで±15分ずらす                             | 🟡 重要   |
| **スパム判定**                 | 投稿間隔を最低4時間。リンク含有率30%以下                    | 🟡 重要   |
| **クロスRTの不自然さ**         | 週2回まで。自然な文脈のみ                                   | 🟡 重要   |

### 4-2. コンテンツリスク

| リスク             | 対策                                                       |
| ------------------ | ---------------------------------------------------------- |
| **AI感のある投稿** | スコアリングで「人間味」2点以上必須                        |
| **炎上リスク**     | `safety_checker.py` でNGワード・センシティブ表現を自動排除 |
| **会社情報の漏洩** | NGワードに会社関連キーワードを登録                         |
| **情報の誤り**     | ファクトチェックプロンプトを組み込み                       |

### 4-3. 安全チェッカー設計

```python
SAFETY_CHECKS = {
    "ng_words": [
        # 情報商材感を排除
        "不労所得", "月100万", "誰でもすぐ", "裏ワザ",
        "知らないと損", "限定公開", "今すぐDM",
        # 会社関連（絶対NG）
        "cyan", "シアン", "ISAI", "異彩",
        # 個人情報
        "北田", "kitada",
    ],
    "max_hashtags": 3,
    "max_links": 1,
    "min_length": 40,
    "max_length": 280,
    "min_human_score": 7,
    "duplicate_threshold": 0.8,      # 過去投稿との類似度80%以上は却下
    "posting_interval_min": 240,     # 投稿間隔は最低4時間（分）
    "daily_limit_per_account": 5,
    "cross_promo_weekly_max": 2,
}
```

### 4-4. 段階的自動化（フェーズ移行条件）

| フェーズ              | 期間    | 自動化レベル                      | 移行条件                       |
| --------------------- | ------- | --------------------------------- | ------------------------------ |
| **Phase 1: 手動承認** | 1-2週目 | 生成は自動、投稿は手動承認        | 投稿案の採用率80%以上          |
| **Phase 2: 半自動**   | 3-4週目 | スコア8+は自動投稿, 7以下は要承認 | 凍結ゼロ, エンゲージメント安定 |
| **Phase 3: 全自動**   | 5週目〜 | 安全チェック通過で全自動投稿      | 1ヶ月間トラブルゼロ            |

```

├── 投稿: 月1,500件まで ← 3投稿/日 × 30日 = 90件/アカウント（余裕）
```

## 8. PDCAサイクル設計

### 8-1. サイクル概要

| 頻度         | 内容                                  | 実行方法                            |
| ------------ | ------------------------------------- | ----------------------------------- |
| **毎日**     | 投稿案生成 → Discord通知 → 投稿       | 自動（GitHub Actions）              |
| **毎日**     | エンゲージメント取得                  | 自動（23:00）                       |
| **毎週月曜** | 週次分析レポート → マスターデータ更新 | 自動生成 → Discord通知              |
| **毎月1日**  | 月次振り返り → 戦略調整               | 半自動（レポート自動 → 判断は手動） |

### 8-2. 週次分析レポートの内容

```
# 週次レポート — 2026/02/16 〜 02/22

## 全体サマリー
- 投稿数: 21本（目標21本 → 達成率100%）
- 平均いいね: 23.4（先週比 +12%）
- ベスト投稿: 「GASで...」（いいね 87）
- フォロワー増減: +42

## タイプ別パフォーマンス
| タイプ | 投稿数 | 平均いいね | 平均RT | 先週比 |
|--------|--------|-----------|--------|--------|
| 問題提起 | 4 | 34.2 | 5.8 | +18% |
| How to | 5 | 28.7 | 8.2 | +5% |
| ストーリー | 4 | 41.3 | 3.1 | +22% |
| ...

## 改善アクション（来週）
1. ストーリー型が好調 → 来週は比率を25%→30%に
2. リスト型が低迷 → フック部分を「数字×疑問形」に変更してテスト
3. 21時投稿が安定的に高いため、21:30→21:00に時間変更

## マスターデータ自動更新
- [追加] 伸びるフック: 「まだ○○してる人、マジで...」
- [除外] 伸びないパターン: 「○○の5つのメリット」（リスト型の平凡なフック）
```

### 8-3. マスターデータの自動進化

```
週次サイクル:
  ① 過去7日の投稿パフォーマンスを取得
  ② エンゲージメント上位3投稿の共通パターンを抽出
  ③ 下位3投稿の失敗要因を分析
  ④ マスターデータに「勝ちパターン」を追記
  ⑤ 翌週の投稿生成プロンプトに自動反映
```

## 9. 横展開テンプレート（アカウント②③追加時）

アカウント①が完成したら、以下を作るだけで横展開できる:

```
追加ファイル:
├── src/generate/templates/account_2.md    ← ②用のプロンプト
├── src/generate/templates/account_3.md    ← ③用のプロンプト
├── data/master/account_2_master.md        ← ②用のマスターデータ
├── data/master/account_3_master.md        ← ③用のマスターデータ
└── config/accounts.json                   ← アカウント情報を追記

設定変更:
├── .env に各アカウントのOAuth認証情報を追加
├── Discord に #account-2-posts, #account-3-posts を追加
└── GitHub Actions のワークフローに追加（or 1つのワークフローでループ処理）
```

**やること**:

1. 新Xアカウント作成（メール・電話番号紐付け）
2. マスターデータ作成（ペルソナ・トーン・NG表現を定義）
3. プロンプトテンプレート作成
4. OAuth認証情報取得
5. config/accounts.json に追記
6. → あとはシステムが自動で回す
